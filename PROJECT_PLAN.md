# 中国调休智能闹铃 App 项目计划

更新时间：2026-05-12

## 当前定位

`CNAlarm` 是 iOS 26+ SwiftUI App，用于把中国法定节假日、调休补班、企业特殊工作制和个人临时安排合并到闹铃计划中。主闹铃通过 AlarmKit 注册到系统，提醒与确认通过本地通知和应用内提醒中心承载。

## 已实现范围

- 普通闹铃：支持标题、时间、启用状态、智能工作日、稍后提醒、铃声选择与试听。
- 闹铃组合：支持基于目标时间 DDL 展开多个子闹铃，模板可复用，闹铃可复制模板后单独覆盖。
- 默认闹铃：新安装后预置 3 个关闭状态的闹铃：`08:10` 起床闹铃（DDL 倒推组合）、`08:57` 门禁打卡（普通闹铃）、`13:57` 午休结束（三段叫醒组合）；用户启用前不会注册到系统闹铃。
- 组合模板：默认预置“三段叫醒”“DDL 倒推”“密集叫醒”。支持普通等间隔模式，也支持复杂模式；复杂模式提供密集叫醒预设、常用时间点添加、每一响独立加减；达到相邻响铃或 0/240 分钟边界时按钮置灰，不自动删除该响。
- 新建组合闹铃：添加闹铃时可直接选择组合模板，模板会同步复制锚点、偏移、默认铃声与稍后提醒设置。
- 中国调休日历：支持国家节假日/补班、手动例外、企业规则和个人休假共同参与工作日识别。
- 休假板块：寒假、暑假已从中国特色工作制中剥离，和年假、婚假、产假/陪产假、病假、事假、调休假一起放入独立“休假”设置；通过弹出日历点选或按住跨日期滑动多选，不再使用单独开始/结束日期选择，旧寒暑假开关逻辑不再作为入口。
- 首页日历：显示本周与下周闹铃概览，点击日期可设置当天上班、当天休息、跳过闹铃、新增临时闹铃；日期格会用独立标记区分节假日休息、调休补班、企业加班日和额外休息日。
- 未来预览：从首页日历打开，从本周开始一直预览到本年度结束；按月份分区、按周折叠，本周默认展开，其他常规周默认收起，存在节假日/补班/企业加班/额外休息标记的周默认展开。
- 连续休息预览：只要出现 `假` 或 `休` 这类连续休息日并跨周，就从涉及休息的第一周扩展到最后一周，合并成一个按时间顺序插入月份内的完整日历框；如果个人休假前后紧挨自然周末或法定节假日，也把这些桥接休息日纳入同一段连续假期。寒暑假、国庆、跨周年假等不再拆成多个周卡，也不置顶。区块内部区分“实际休息日期”和“展示用完整周日期”，标题显示实际休息天数，日历框展示完整周。
- 日期标记：`假` 表示国家节假日，`班` 表示调休补班，`加` 表示月末周六/大小周/单休等企业加班日，`休` 表示个人休假或手动额外休息；标记贴合在日期圆圈底部，不再与圆圈分离。国家节日和额外休假会在日期格内显示具体名称，周卡标题摘要也显示端午节、中秋节等具体名称，方便在未来预览中直接看出节日/假期来源。
- 无干扰日期：休息、休假或当天没有闹铃时，不显示灰色占位小点；无闹铃的未来预览卡片收紧高度，避免出现大块空白。
- 加班覆盖：月末周六、大小周、单休等企业加班规则遇到国家节假日或调休补班日时，由国家日历标记覆盖，不再叠加显示企业加班标记。
- 提醒中心：生成周五晚检查、节假日前提醒、调班前一晚提醒；同类提醒仅展示未来少量关键项。
- 声音：提供 App 内置铃声、导入铃声和 AlarmKit 系统默认占位；AlarmKit 系统默认声由系统播放，不做 App 内试听承诺。
- Apple Watch：不做独立 watchOS target，依赖 AlarmKit 系统转发和 iPhone 通知镜像。

## 规则优先级

日历判定优先级保持为：

1. 手动 `CalendarException`
2. 国家节假日/调班数据
3. 企业规则
4. 默认周一至周五

闹铃调度由 `AlarmScheduleResolver` 统一展开普通闹铃、组合闹铃和临时闹铃，再交给同步服务注册系统闹铃。

## 当前重点问题与修复策略

- 未来预览首次打开为空：改为打开前生成快照，并用 `sheet(item:)` 传递，避免 sheet 读取旧状态。
- 组合模板删除卡顿：删除前弹出确认，确认后清理引用并保存，避免列表 swipe 删除直接触发较重的 SwiftData 更新。
- 组合模板表达能力不足：复杂模式支持非等间隔时间点，并通过预设与边界锁定降低编辑成本。
- 未来预览性能：生成预览时复用一次解析后的节假日数据，普通周只显示压缩摘要行，用户展开时才渲染日历详情；跨月周的展开状态按月份隔离，连续跨周休息日期从周列表剥离为范围卡。小日历使用固定行列布局和稳定高度，避免 `LazyVGrid` 在长列表滚动时产生高度回弹、底部裁切和上滑抖动。

## 测试计划

- 单元测试：节假日解析、工作日优先级、企业规则、日期标记分类、默认闹铃与模板种子、个人休假连续范围合并、国庆等法定节假日跨周范围扩展、组合展开、非等间隔组合、提醒生成、过期例外隐藏。
- 构建验证：每次影响 UI 或模型后至少运行 iOS Simulator build。
- 真机验证：后续每次完成代码或 UI 修改后，跑完最快相关验证，再同步安装到手机确认。

## 后续维护约定

后续每次修改功能、规则、调度策略、UI 信息架构或已知边界时，同步更新本文件，让计划始终反映当前版本；每次改完也同步安装到手机，保持真机状态和代码一致。
